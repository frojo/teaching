Week 4 (Wednesday) - Physics

Unity has a very robust physics system built into it. You can make use it to make games that play with physics or have some sort of physics simulation in them. Let’s learn how to do physics stuff in Unity!

First:

What do games use physics for?

  • bouncing objects
  • falling objects
  • gameplay mechanics
  • realism, interactivity
  • characters interacting with objects
  • water physics

What do games use colliders for?

  • detecting when objects touch
  • hitboxes
  • for triggering loading, checkpoints, cutscenes etc.
  • for more realistic experiences

2D vs 3D

Unity has two physics systems. One for 2D physics, one for 3D physics. For the most part, these systems work very similarly, each having their own components. If you learn how to do something in one system, it probably works in the other one.

2D physics and 3D physics can’t interact with one another.

Colliders

To be a part of the physics system, an object needs a Collider. There are many shapes of colliders, both in 2d and 3d. All Collider shapes are essentially the same, except for the shape they construct.

How to add a collider to a GameObject

Colliders are components

  • Click on the object you want to add a collider to and click Add Component. Then search for “collider"

Unity will add a collider for you automatically when you create a 3d object.

How to choose the right collider

There’s a lot of options! Over time, you’ll know which collider is appropriate for what you want. When you’re choosing what collider to use, some things to consider:

  • If you’re working in 2D, use a 2D collider

  • What general shape is your sprite? Try to use the simplest collider shape that still makes sense for your sprite. For most things, you can usually use Box or Circle

Editing the collider

Now that you have a collider on your GameObject, select it. You should be able to see the collider as a thin green outline. You can click Edit Collider to manually edit it (usually, Unity should make the collider the right size automatically though)

Rigidbody

For a GameObject to be a part of the physics system, it also needs a Rigidbody component.

Add a Rigidbody in the same way as you add any other component: select your object and click Add Component in the inspector

Basic Rigidbody options

Mass: Control how heavy the object is

Drag: Making this higher will make it so the object is flowing through water instead of air

Use Gravity: Whether or not you want this object to be affected by gravity

Is Kinematic: Turns this off to create an object not affected by physics (e.g. walls, ground, static things)

Constraints: Limit the movement of the object in some ways!

Note: Rigidbody2D works a little different, but it’s mostly the same

Physics Material

Change how the Game Object will be affected by the collision…controls how bouncy things are for example.

Note: This is different from the Material that we learned about earlier

Colliders and code: OnCollisions and OnTriggers

In code, we can use Colliders for two different sorts of events: Collisions and Triggers

  • Collisions are when two objects physically bonk into eachother. Collisions use the Unity physics system. Collisions don’t work super well when we move objects ourselves with code (as we’ve been doing so far). Think physics-y games like Angry Birds. This is what colliders do by default
  • Triggers are when we use colliders to activate things or make things happen when two object overlap. To enable this, you check the Is Trigger box in the collider

Let’s look at Collision events first

Using Collisions in code

There are 3 Collision functions you need to know. These are special built-in functions in Unity. You need to include these special functions on a script that IS CURRENTLY ON the Object you want to track.

OnCollisionEnter is called the first frame your collider or rigidbody begins touching another rigidbody or collider. OnCollisionStay is called every frame they’re touching, and OnCollisionLeave is called the frame they are no longer touching.

public class CollisionExample :MonoBehaviour
{
    //Moves thisGameObject 2 units a second in the forward direction
	  void Start()
    {
    }  

		void Update()
    {
    }

    // Called when this object starts colliding with anothre object
    private void OnCollisionEnter(Collision other)
    {
        Debug.Log("First frame of collision with another object");
    }

		// Called *while* this object is colliding
		 private void OnCollisionStay(Collision other)
    {
        Debug.Log("Currently still colliding with another object");
    }

	// Called when this object stops colliding with anothre object
		private void OnCollisionExit(Collision other)
    {
        Debug.Log("We just stopped colliding with another object");
    }
}

What is Collider other ? This is basically the other object that is being collided with.

Using Triggers in code

You can also use colliders as *******Triggers. This means that the objects won’t bonk/collide into each other — they’ll pass through each other. But you can still use these colliders to trigger code when colliders are overlapping

There are special OnTrigger functions that are called in the event that two objects with colliders overlap

OnTriggerEnter is called the first frame your collider begins overlapping another collider. OnTriggerStay is called every frame they’re overlapping, and OnTriggerLeave is called the frame they are no longer overlapping.

Important: Make sure you check Is Trigger for these functions to be called

public class Example :MonoBehaviour
{
    //Moves thisGameObject 2 units a second in the forward direction
	  void Start()
    {
    }  

		void Update()
    {

    }

		private void OnTriggerEnter(Collider other)
    {
        //Debug.Log("Entering the cube!");

				Camera.main.enabled = false;
    }
   

		private void OnTriggerStay(Collider other)
    {
        Debug.Log("In the cube!");
    }

	// Called when this object stops overlapping with anothre object
		private void OnTriggerExit(Collider other)
    {
        Debug.Log("Just left the cube!");
    }
}

2D OnCollisions and OnTriggers

Both the OnCollision... and OnTrigger... functions have their 2D versions that you must use if you’re doing physics in 2D.

They’re the same functions but with 2D at the end of them. **Andi ******************************** also use Collider2D instead of Collider.

E.g.:

		 private void OnTriggerEnter2D(Collider2D other)
    {
        Debug.Log("First frame of overlap with " + other.name);
    }
