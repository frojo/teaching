Gitignore and Gitattributes

  • .gitIgnore is a file that specifies which files should not be
    • Basically, some files are generated automatically in the backend of unity, so committing them to source control is a waste of time at best, and can cause mountains of unnecessary conflicts at worse.
  • • .gitattributes is a file that specifies which file types should be added to GIT LFS

Git LFS

LFS stands for Large File Storage

  • Needed to be initialized when dealing with large files.
  • Kinda a pain.
  • The files that use LFS are written in the projects .gitattributes

Making Commits and Pushing

  • Everytime you make a new commit
    • Write a description of what’s changed
    • Commit to save the changes
    • Push the changes to make the changes visible to everyone else / different machines.
  • Before starting work, you will want to “pull” changes, so your files are up to date.
