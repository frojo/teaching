Physics movement

I’ve been saying that you can’t move physics objects with code like we did for the flatgame. Well, there are still ways to move physics objects.

The way you move physics objects is by adding forces that push on them. You can’t just move it, you have to push

Let’s see how!

Setting the scene

There’s two objects, inspect them.

  • Dynamic Circle
  • Static floor

Let’s add a script to this object. We can call it PhysicsMove.

What code do we write to let us move this object?

If you want to move a physics object with code, you need to have the Rigidbody component, so let’s get that first.

Accessing the Rigidbody

You get the rigidbody like you would any other component:

public class PhysicsMove :MonoBehaviour
{

		Rigidbody2D myRigidbody2D;

	  void Start()
    {
				myRigidbody2D = GetComponent<Rigidbody2D>();
    }  

		void Update()
    {
    }

 
}

Moving your object

There are a few different ways of moving a physics object.

Velocity

You can move your object by setting a velocity.

myRigidbody2D.velocity = new Vector3(1, 0, 0);

Adding a Force

You can move your object by setting a velocity.

myRigidbody2D.AddForce(new Vector3(1, 1, 0), ForceMode2D.Force));

These two options both feel different. Try writing some code that will do one of these when you press a key, and then try the other option and see how it feels different.

FixedUpdate

There’s a special function called FixedUpdate()

It’s like Update(), but it happens at a fixed interval instead of a variable interval.

FixedUpdate() is important for physics because you should do your physics movements in FixedUpdate() (instead of Update())

public class PhysicsMove :MonoBehaviour
{

		Rigidbody2D myRigidbody2D;

	  void Start()
    {
				myRigidbody2D = GetComponent<Rigidbody2D>();
    }  

		void Update()
    {
				// don't do physics code in here!!
    }

		void FixedUpdate()
    {
        if (Input.GetKey(KeyCode.D)) {
            myRigidBody2D.AddForce(new Vector3(1, 1, 0), ForceMode2D.Force);
        }
    }

 
}

Joints

Joints let you connect two more different objects together in ways that mirror common real-world connections between physical objects. There’s a bunch of different joints!

Fran’s attempt at a little explanation for the most common joints:

Fixed joint: Like having a rigid metal rod connecting two objects

Hinge joint: Like how a door moves on its hinge

Spring joint: Connecting two objects with a spring

Slide joint: Like the two objects are connected by a trumbone slide that is infinitely long

How to add a joint

If you have two objects and want to connect them with a joint:

  • Select one object and click Add Component
  • Search for Joint
  • Make sure you pick the 2D joint for 2D etc.

Joints are kind of complicated and may not work the way you expect them to at first. The best thing to do is play with it! And ask me questions.
