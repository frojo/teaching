2D vs 3D mode in Unity

Basics of “working to 2d mode”

  • To put your Unity in 2D mode, click on 2D in the Scene view
  • Set your Main Camera to Orthogonal (not Perspective)
    • Now if you want to change the size of the camera viewport, you change “Size” in the Camera settings

Tip: if you want a sprite to be “on top of” another sprite, make it’s z positoin less than that sprite’s z position

Text

Creating a Text GameObject

  • Right click in the Hierarchy: 3D Object > Text - TextMeshPro

    This should prompt you to import TextMeshPro. Click to import TextMeshPro Essentials

Basics of the Text Component

Select your new text gameobject and look in the Inspector View at the TextMeshPro - Text component

  • There’s a Text Input box for changing the displayed Text
  • You can change the Font (we’ll discuss how to do that later)
  • You can make it bold, italic etc.
  • You can change the color of the text, and even have a gradient
  • and a bunch of other stuff

Custom font

In Unity, you can’t simply import a font file and use it. You have to also create some TextMeshPro assets from your font file

  • Get the font file for the font you want to use. It should be a .otf or .ttf file. We’ll use this example:

    YoungSerif-Regular.otf

  • Import the font file into your Unity project (drag it in)

  • Open the TextMeshPro Font Asset Creator by going to Window > TextmeshPro > Font Asset Creator

  • Drag in your font asset into the Source Font File field

  • Click Generate Font Atlas

  • Click Save and save it as the default name

Now you’ve created a new font asset that you can use!!

  • In your Text object, you can change the font to your new font by clicking Font Asset

Audio

How to add music to your game

  • Find a .mp3, .wav or some other sound file and import it into your Project (by dragging it in)

    01 - Mahogany Hall Stomp - Louis Armstrong And His Orchestra-Restored.mp3

  • Once it’s in your Unity project, you can drag it from you Project view into your Scene and it’ll automatically create a GameObject with the song

  • The most important options are Mute, Play on Awake and Loop

You can control audio with a script by doing something like:


using UnityEngine;

public class Example :MonoBehaviour
{
		AudioSource myAudioSource;

    void Start()
    {
        //Fetch theAudioSource from theGameObject
        myAudioSource = GetComponent<AudioSource>();
       
    }

    void Update()
    {
				if (Input.GetKeyDown(Keycode.Spacebar)) {
					myAudioSource.Play();
				}

				if (Input.GetKeyDown(Keycode.S)) {
					myAudioSource.Stop();
				}

    }

Colliders and triggers

Colliders is how you get objects to… collide with each other!

First, we can use Colliders for two different things: Collisions and Triggers

  • Collisions are when two objects physically bonk into eachother. Collisions use the Unity physics system. Collisions don’t work super well when we move objects ourselves with code (as we’ve been doing so far). Think physics-y games like Angry Birds
  • Triggers are when we use colliders to activate things or make things happen when two object overlap

We’re just going to be use colliders for Triggers today

How to add a collider to a GameObject

Colliders are components

  • Click on the object you want to add a collider to and click Add Component. Then search for “collider"

How to choose the right collider

There’s a lot of options! Over time, you’ll know which collider is appropriate for what you want. When you’re choosing what collider to use, some things to consider:

  • If you’re working 2D, use a 2D collider

  • What general shape is your sprite? Try to use the simplest collider shape that still makes sense for your sprite. For most things, you can usually use Box or Circle

Collider basics

Now that you have a collider on your GameObject, select it. You should be able to see the collider as a thin green outline.

  • Edit Collider

Using colliders as Triggers

  • both objects must have colliders
  • Check Is Trigger on the one with the script

Using Triggers in code

There are special OnTrigger functions that are called in the event that two objects with colliders overlap

OnTriggerEnter2D is called the first frame your collider begins overlapping another collider. OnTriggerStay2D is called every frame they’re overlapping, and OnTriggerLeave2D is called the frame they are no longer overlapping.

Important: Make sure you check Is Trigger for these functions to be called

public class Example :MonoBehaviour
{
    //Moves thisGameObject 2 units a second in the forward direction
	  void Start()
    {
    }  

		void Update()
    {
    }

    // Called when this object starts overlapping with anothre object
    private void OnTriggerEnter2D(Collider2D other)
    {
        Debug.Log("First frame of overlap with " + other.name);
    }

		// Called *while* this object starts overlapping with anothre object
		 private void OnTriggerStay2D(Collider2D other)
    {
        Debug.Log("Currently still overlapping with " + other.name);
    }

	// Called when this object stops overlapping with anothre object
		private void OnTriggerExit2D(Collider2D other)
    {
        Debug.Log("We just stopped overlapping with " + other.name);
    }
}
