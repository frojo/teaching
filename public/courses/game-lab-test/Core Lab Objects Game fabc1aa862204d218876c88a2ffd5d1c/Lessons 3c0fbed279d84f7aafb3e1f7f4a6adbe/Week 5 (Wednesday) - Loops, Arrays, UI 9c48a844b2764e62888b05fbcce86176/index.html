For loops

Sometimes we want to do something many times. Over and over again.

We use loops to do this.

For loop

Use the for loop to specify having an action occur a set number of times.

for (int i = 0; i < 4; i++) {
    Debug.Log("Hello!");
}

There are 3 sections in the for loop statement. First we initialize the temporary variable. Then we test to see if the statement is valid. Then we increment the temporary variable at the end of each loop.

Another example, this time using our temporary variable

for (int i = 0; i < 100; i++) {
  Debug.Log(i+" bottles of beer on the wall, "+i+" bottles of beer...");
}

Conclusion: For loops give us the power to repeat things tons of times. It’s one of the superpowers of the computer.

Note: different from the Update() loop!

Update() is also a function that “loops”. But each pass of Update() happens about 1/60th of a second apart from each other. That’s a small amount of time for humans, but for computers that’s a lot of time.

For loops, on the other hand, happen a lot faster, basically instantaneously.

When to use an if statement or for loop

If you are testing whether something is true

When you are asking a question, use an if statement.

Examples: Is my character off screen? Is the color black?

If you need to make lots of things, use a for a loop

Arrays

An array is a list of similar objects, usually in a row or column.

Arrays have length. The length equals the total number of values in the array.

How to initialize an array:

// this initializes an array of 4 "empty" values 
// in this case, this will be 4 0s
int[] myArray = new int[4];  

// an array with the values 1, 2, 3, 4
int[] myArray = {1, 2, 3, 4};

string[] students = {"Lizzy", "Gabriel", "Frida",
                     "Xiona", "Lauren", "Amal",
                     "Malika", "Xinyi", "Xiara",
                     "Lily", "Eve", "Tyne"};

The length of students is 12 since there are 12 entries.

Debug.Log(students.Length); // 12

The first element is at index 0.

Debug.Log(students[0]); // Lizzy

Counting up from 0, we can see that the last element in the array is the 4th.

Debug.Log(students[4]); // Lauren

Extrapolating from here, this tells us that the last element in any array is length - 1.

Debug.Log(students[students.Length - 1]);

Arrays and loops together forever

If loops let us do something many times, and arrays are a list of many things… hold on, what if we use them together?

Arrays and loops are best friends.

Let’s say we wanted to print out all our students’ names. Well we could do this:

Debug.Log(students[0]);
Debug.Log(students[1]);
Debug.Log(students[2]);
Debug.Log(students[3]);
...
Debug.Log(students[11]);

Or! we could replace those 12 lines of code with:

for (var i = 0; i < students.Length; i++){
    Debug.Log(students[i]);
}

UI Basics

Sometimes when making a game, you want to communicate certain information to the player, but it’s tricky to convey that information in the world of the game. That’s where UI comes in!

Screenspace vs Worldspace

UI exists on literally a different plane as the game world. We often call this UI-land the screenspace in contrast to worldspace.

If you know film terminology, it’s helpful to think of the UI as undiagetic. In other words, not a part of the world of the game, but outside of it.

One thing about UI is that it’s really deep! Slick menus, good-looking textboxes take time and ingenuity to design and implement.

The Canvas

The Canvas is like a screen that is overlayed on top of your game world. All your UI elements will go in your Canvas

How to create the Canvas

Left Click Hierarchy -> UI -> Canvas

This will also create an object called EventSystem. Don’t delete that!!!

How to create UI Text

Hierarchy left click -> UI -> Text ( TextMeshPro)

This is a similar but different thing to the text that we created for our flatgames!!

Remember, UI text is in screenspace while the other text was in worldspace

Rect Transform

Objects in your Canvas have a special Transform called the Rect Transform. This is a different way of tracking position/size etc. and it makes it easier for working with UI elements.

Anchors let you control how the UI will change based on the Aspect Ratio of the screen (e.g. a square screen vs a widescreen)

Anchors are part of what makes UI hard!

Anchors take a bit to get used to, but keep playing with it to build an intuitive understanding

Changing text through a script

  1. Include the TextMeshPro and Unity UI libraries at the top of your file
using TMPro;
using UnityEngine.UI;
  1. In your class, declare the Text component (familiar?) and assign it in the Inspector, and then use it like this
// Make sure to assign this in the inspector!!
public TextMeshProUGUI changeThisText;

//...... later on in the code

changeThisText.text = "this is some new text!";
