Rube Goldberg Contraption | DUE 9/29

▶️ IN UNITY: Create a "Rube Goldberg Contraption" that plays out a sequence of successive actions using scripts and physics.

Step 1: Initial Prototype

  • build a scene where (1) a ball rolls down a ramp, then (2) bounces off the floor, and then (3) hits some dominoes on a platform
    • (remember: to make something bouncey, you must set the ball to use a Physic Material with Bounciness = 1 and Bounce Combine = Maximum.)
  • test to see if it works
  • once it works in the editor, build out to WebGL and see if it works in your browser (make sure you click "Build and Run")
    • (WHY? if there's problems with the build, you'll want to find out as soon as possible)

Step 2: Camera Triggers

  • create an invisible cube trigger object
  • tag every Camera as "MainCamera"
    • double-check that your Main Camera is tagged as "MainCamera"
    • create a Camera object, and also set its tag to "MainCamera", and then disable it (uncheck the checkbox on the Camera component)
  • translate this pseudocode, and put the script on the trigger:
declare a public variable of type Camera called "myCamera"

OnTriggerEnter:
	set "enabled" on Camera.main to false
	set "enabled" on myCamera to true
  • test the script, see if you can trigger a camera change when the ball bounces off the floor
  • now that you have a working camera trigger, you can just duplicate this gameObject to make a new one!

Step 3: (Optional) Camera Improvements

This isn't required, but if you want, code some additional functionality for your camera switching system...

  • to make the camera follow an object, translate this pseudocode and add it to the camera script:
declare a public variable of type Transform called "followTarget"

Update:
	if followTarget is not equal to null, then:
		// what's the formula to get a vector from point A towards point B?
		declare a variable of type Vector3 called "newForwardDirection", and set it equal to: the vector from myCamera's transform's position towards followTarget's position
		set myCamera's transform's forward equal to newForwardDirection
  • to visualize the camera triggers in the Unity Editor, translate this pseudocode and add it to the camera script:
// OnDrawGizmos is a special function that lets you draw helper gizmos in the Scene view
OnDrawGizmos:
	// google "ondrawgizmos draw wire cube"
	draw a wire cube at this transform's position, with this transform's localScale
	
	if myCamera is not equal to null, then:
		// google "ondrawgizmos draw line"
		draw a line from this transform's position to myCamera's position

Step 4: Plan your Machine

Spend 5 minutes to sketch (on paper) a plan for your rube goldberg machine

  • feel free to watch rube goldberg machines on YouTube for inspiration
    • remember Unity's physics limitations: no soft bodies or ropes or cloths
  • don't just build in the editor randomly, it will take longer! but also don't wait too long before building in the editor
  • sketching is part of the design and development process
  • where will your different cameras and camera triggers go?
  • Need ideas on what to add to your machine? Check the brainstorm doc we made!

Step 5: build your machine

  • test often!
  • when you are halfway done, do a test build in WebGL... trust me
  • hint: if you need to test the second half of your machine, you'll have to watch the first half of your machine over and over... unless you made a cheat code that could teleport a ball to a special position?
  • The whole 'machine' should have around 7 phases / stages
  • you must have at least 3 camera changes, using physics triggers
  • No input beyond starting the contraption
  • there should be some kind of finale, activated by a physics trigger
  • BONUS: add school-appropriate music, no OK Go
  • build out to WebGL, upload to itch.io, and submit it on Canvas
